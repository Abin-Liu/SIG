开发接口

    作为一个通用型GUI，SIG为其它模块或脚本提供了一系列开发接口，以使它们能够和GUI之间进行必要的交流，同时也得以将必要的信息及时以直观的方式发送给用户。目前笔者公开的命令有以下几项。

PostString

功能 从游戏内向SIG主界面发送文字信息
命令 SIG PostString <文字内容>
效果 将 <文字内容> 显示在SIG程序主界面的信息条中，支持中文字符。
使用举例 在SIG程序主界面的信息条中显示"找到一把Windforce啦!"
D2Hackit模块开发者 server->GameCommandLine("SIG PostString 找到一把Windforce啦!");
D2JSP脚本开发者  <暂时无法支持>

Next...命令组

功能 预设下一个游戏的状态(创建/加入、游戏名、密码)，在当前游戏结束后，SIG将会以指定的方式创建或加入下一个游戏。
命令 SIG NextGame <create/join>
SIG NextName <game name>
SIG NextPassword <game password>
　
使用举例1：下一个游戏加入名叫"abcde"，密码为"123"的游戏
D2Hackit模块开发者 server->GameCommandLine("SIG NextGame join"); // 类型为join
server->GameCommandLine("SIG NextName abcde"); // 游戏名为abcde
server->GameCommandLine("SIG NextPassword 123"); // 密码为123
D2JSP脚本开发者 <暂时无法支持>
　
使用举例2：下一个游戏创建名叫"pkpk"的无密码的游戏
D2Hackit模块开发者 server->GameCommandLine("SIG NextGame create"); // 类型为create
server->GameCommandLine("SIG NextName pkpk"); // 游戏名为pkpk
server->GameCommandLine("SIG NextPassword "); // 密码为空
D2JSP脚本开发者 <暂时无法支持>
